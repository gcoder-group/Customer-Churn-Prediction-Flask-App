1. Problem Statement

Customer churn refers to the percentage of customers who stop using a company's product or service over a period of time.

For subscription-based businesses such as telecom companies, customer churn directly impacts:

  • Revenue
  • Customer lifetime value
  • Business growth

Goal: Predict whether a customer is likely to churn, allowing the company to take proactive retention actions.

2. Dataset Description

Source: Telco Customer Churn Dataset

Total Records 7,044
Input Features 20
Target Column Churn
Data Type Structured Tabular Data
3. Understanding the Features
Customer Profile
  • Gender
  • SeniorCitizen
  • Partner
  • Dependents
Service Information
  • InternetService
  • PhoneService
  • StreamingTV
  • OnlineSecurity
Billing Information
  • MonthlyCharges
  • TotalCharges
  • PaymentMethod
  • Contract Type
Customer History
  • Tenure (months stayed)
4. Data Preprocessing
Handling Missing Values
  • TotalCharges column had missing values
  • Converted to numeric format
  • Missing values handled using median imputation
Handling Categorical Features
  • Many features are categorical (Contract Type, Internet Service, Payment Method, etc.)
  • Converted to numeric using One Hot Encoding
  • Example: Contract Type → Month-to-month, One Year, Two Year
5. Feature Engineering Pipeline

A Scikit-Learn Pipeline was created to automate preprocessing and model training.

Numeric Features Processing
  • Missing value imputation (median)
  • Feature scaling (StandardScaler)
Categorical Features Processing
  • Missing value imputation (most frequent)
  • One-hot encoding
Advantages of using Pipeline:
  • Ensures consistent preprocessing
  • Prevents data leakage
  • Simplifies model deployment
  • Allows easy integration with APIs
6. Train-Test Split
Training Data 80%
Testing Data 20%
Sampling Method Stratified Sampling

Why stratification? Ensures both sets maintain the same proportion of churn and non-churn customers, preventing biased datasets.

7. Model Selection
Algorithm: Logistic Regression

Why Logistic Regression?

  • Works well with structured data
  • Highly interpretable
  • Computationally efficient
  • Suitable for binary classification problems

Output: 0 → Customer will stay, 1 → Customer will churn

8. Model Deployment

The trained model was deployed using Flask with REST APIs:

API Endpoint Description
/predict_api Returns churn prediction, probability, and actual result
/sample/ Returns random sample customers for testing
/model_metrics_api Returns model performance metrics
9. Business Impact

Customer churn prediction enables companies to:

  • Identify high-risk customers
  • Take proactive retention actions
  • Reduce customer loss
  • Increase revenue and customer lifetime value
